Kubebuilder is a framework used to write a specific, custom operator from scratch in Go.

Crossplane is an engine that allows you to build a Custom Control Plane purely using declarative configuration (YAML Compositions), without writing a single line of Go.


kubebuilder-operator -> Custom code-first compiled binary.
crossplane-config -> Configuration-first engine composition layout

-----controller logic

The operator needs to:
1)Fetch the incoming DataStream event.
2)Connect to our internal Kafka cluster (kafka.kafka.svc.cluster.local:9092).
3)Create the topic if it doesn't exist.
4) Write out a ConfigMap containing connection coordinates.
5)Spin up a basic Deployment acting as our automated consumer app.

-------------------RBAC
The project requires mixed orchestration (Kafka Topic + ConfigMap + Deployment). In Crossplane, Topic is reconciled by provider-kafka, while ConfigMap/Deployment are reconciled by provider-kubernetes. Therefore provider-kubernetes must have namespace permissions through Role/RoleBinding. Without RBAC, the DataStream pipeline cannot reach Ready, so the self-service abstraction is incomplete.
